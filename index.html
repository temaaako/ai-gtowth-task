MVP или AI-проект, который я сделал

Project Man AI

Персональный AI-проджект-менеджер в Telegram

Что это за продукт и какую проблему он решал

Project Man AI — это Telegram-бот, который работает как персональный проджект-менеджер с настраиваемым характером. Идея родилась из личной боли: когда работаешь на себе или удалённо, некому контролировать задачи и дедлайны. Таск-трекеры — пассивные списки, они не пинают. Project Man решает это: бот живёт в чате, у него есть характер (от мягкого ментора до жёсткого руководителя), и он сам выбирает момент, чтобы написать и спросить про прогресс. Продукт именно для людей, которым важно, чтобы над ними кто-то стоял и требовал, даже когда работа самостоятельная.

Бот сам пишет: «Ты обещал закончить макет к пятнице. Сегодня четверг. Как дела?» Если срываешь дедлайн — нужно объяснить, что пошло не так. Плюс я заложил интеграцию с Jira и Trello через API, чтобы бот подтягивал реальные задачи.

Стек

Python / aiogram Antigravity LLM API Jira / Trello API SQLite OpenClaw

Сроки

Рабочий прототип собрал за 1 день вайб-кодингом. Бот умел принимать задачи, назначать дедлайны и отвечать в характере. Ещё пару дней ушло на доработку логики уведомлений и промптов для разных «персонажей».

Сложности

Управление контекстным окном — нужно было, чтобы бот помнил историю задач, но не вылетал за лимит токенов. Решил через суммаризацию старых сообщений. Параллельно пришлось подбирать оптимальное количество токенов на запрос, чтобы не разориться на API — баланс между качеством ответов и стоимостью.

Отказоустойчивость: API нейронок периодически отваливается, и если бот в этот момент должен был напомнить про дедлайн — пользователь ничего не получит. Пришлось закладывать фолбэк на другую модель, ретраи и очередь отложенных сообщений.

Ну и баланс тона в промптах: «жёсткий режим» должен мотивировать, а не бесить — пришлось итерировать.

Выводы

Вайб кодинг реально позволяет выкатить MVP за день, но от прототипа до продукта есть ощутимый разрыв. Самое ценное в продукте — не AI-ответы, а логика «когда и зачем писать пользователю». Это именно продуктовая логика, а не промпт-инжиниринг.

Что нужно доделать, чтобы MVP начал приносить деньги

Подключить платный AI-бэкенд и протестировать на реальной когорте 20–30 человек. Добавить freemium: бесплатно базовый функционал (3 задачи, стандартный персонаж), расширенный по подписке. Провести CustDev, чтобы понять, за какую конкретно фичу люди готовы платить. Запустить аналитику: retention D1/D7, количество выполненных задач в срок vs просроченных.

// Задание 02

MVP: AI-ассистент для соискателей

vJOBivAI
Персональный радар вакансий в Telegram

Ключевая проблема

Вакансии разбросаны по куче источников: hh.ru, сотни Telegram-каналов, Habr Career, LinkedIn, Twitter/X. Следить за всеми вручную — отдельная задача. На hh.ru конкуренция огромная, многие отклики не просматриваются. В Telegram одна и та же вакансия дублируется в 5–10 каналов. Плюс сам процесс поиска стрессовый, люди откладывают отклики.

Это нельзя решить запросом в чат с нейронкой. Нейронка не парсит вакансии в реальном времени, не дедуплицирует и не присылает подборку каждое утро.

Концепция

Telegram-бот — персональный радар вакансий. Мониторит источники, убирает дубли, классифицирует и каждое утро доставляет подборку 5–7 релевантных вакансий. Tinder-механика (свайп «интересно / пропуск») делает процесс быстрым. Геймификация: streak-счётчик дней подряд, бейджи за активность — всё это формирует привычку и превращает стрессовый поиск работы в простой ежедневный ритуал на 3–5 минут.

Проработка образа MVP, логика и дизайн

Telegram-first, минимум UI. Magic Onboarding (AI-профайлинг) — никакого ручного заполнения анкет. Пользователь просто отправляет боту PDF-резюме или ссылку на профиль. LLM сама вытаскивает специальность, грейд, стек и город. Пользователю остаётся только подтвердить данные одним кликом и указать желаемую зарплату. Дальше бот работает сам. Карточка вакансии компактная — всё в одном экране без скролла. Геймификация встроена в процесс: streak-счётчик (сколько дней подряд пользователь открывал подборку), бейджи за активность, вечерние напоминания если пропустил утреннюю подборку.

Пользователь отправляет файл с резюме, ИИ мгновенно формирует поисковый профиль, и пользователь получает первые вакансии сразу. Каждое утро приходит подборка. За пару недель формируется привычка: 3–4 минуты утром. Бот учится на свайпах — если человек постоянно пропускает вакансии без зарплаты, перестаёт их показывать.

Стек

Сроки

С учётом вайб-кодинга — 2–3 недели до рабочего MVP с тестом на бета-пользователях.

// Задание 03

Три AI-агента для повышения продуктивности CEO

📊
Агент 1: Утренний брифинг по метрикам

Что делает: каждое утро собирает ключевые метрики продуктов компании и выдаёт в Telegram структурированный отчёт. Не просто цифры, а контекст: «DAU упал на 12% за 3 дня, совпадает с обновлением v3.2, возможно баг». Сравнивает с целями, подсвечивает аномалии.

Данные
Amplitude, AppMetrica через API, App Store Connect, Google Play Console, внутренние таблицы
Реализация
OpenClaw-агент с Skills под каждый источник. Запуск по расписанию. LLM для интерпретации и summary. MCP-коннекторы. Доставка через Telegram
Польза
CEO начинает утро с готового анализа вместо разбора 5–7 дашбордов. Экономия 30–60 минут в день. Не пропустишь критическое падение
💰
Агент 2: Анализ расходов и оптимизация

Что делает: мониторит финансовые потоки: подписки на сервисы, инфраструктуру, маркетинговые бюджеты. Находит дублирующиеся подписки, неиспользуемые сервисы, аномальные всплески. Раз в неделю отчёт с рекомендациями.

MVP: агент парсит данные о списаниях из загруженных вручную CSV-выписок (чтобы безопасно протестировать гипотезу без доступов к счетам) или подключается к read-only API сервисов, где есть все важные расходы. Затем он сопоставляет эти данные с данными об использовании сервисов (логины сотрудников, активность в инструментах через API). Если подписка оплачивается, но активности по ней нет — помечает как кандидата на отключение. Классифицирует расходы по категориям, показывает динамику, алертит при аномалиях.

Анализ расходов на персонал и их целесообразности (сопоставление затрат с output), мониторинг unit-экономики продуктов (CAC, LTV, ROI каналов), алерты при превышении бюджетов.

Реализация
Python-сервис для сбора данных по расписанию. LLM для классификации и рекомендаций. Claude Agent SDK + MCP-коннекторы. Telegram для отчётов и алертов
Польза
В компании с 100+ сотрудниками расходы легко «размазываются». Агент находит конкретные точки оптимизации
⚡
Агент 3: Командный бот CEO в Telegram

Что делает: OpenClaw-бот в Telegram, через который CEO одним сообщением может поставить задачу сотруднику, отправить email, поставить напоминание, запросить статус проекта, найти информацию в базе знаний компании. Бот не просто пробрасывает команды в рабочие инструменты, а действует как полноценный ассистент: перед тем как назначить задачу, проверяет текущую загрузку исполнителя в трекере и календаре. Если человек перегружен, бот сам предупреждает об этом и предлагает альтернативу — переназначить на другого или сдвинуть дедлайн. То есть CEO не просто делегирует действие, а получает контекст для принятия решения.

Данные
Таск-трекер (Jira/Asana), корпоративная почта, календарь, база знаний (Notion/Confluence), контакты команды
Реализация
OpenClaw Gateway + Telegram. Skills для каждого сервиса. Persistent memory. Мощная LLM для интерпретации свободных команд. Claude Agent SDK для сложных цепочек
Польза
Задачи ставятся за секунды без открытия трекера. Письма отправляются без переключения контекста. Всё логируется в рабочих инструментах команды. Экономия 1–2 часа в день
